## ADDED Requirements

### Requirement: 实例健康状态监控

系统必须监控数据库实例的健康状态，提供实时的健康检查。

#### Scenario: 健康检查初始化

- **WHEN** 系统启动
- **THEN** 必须初始化健康检查组件
- **AND** 必须为每个实例创建健康检查器
- **AND** 必须设置默认的检查间隔

#### Scenario: 定期健康检查

- **WHEN** 健康检查间隔到达
- **THEN** 系统必须执行健康检查
- **AND** 必须尝试连接数据库实例
- **AND** 必须执行简单的 SQL 语句（如 SELECT 1）
- **AND** 必须记录检查结果

#### Scenario: 健康状态更新

- **WHEN** 健康检查完成
- **THEN** 系统必须更新实例的健康状态
- **AND** 状态必须是以下之一：Healthy、Unhealthy、Unknown
- **AND** 必须记录状态变更日志

---

### Requirement: 健康检查配置

系统必须支持健康检查的配置，包括检查间隔、超时时间等。

#### Scenario: 配置检查间隔

- **WHEN** 配置健康检查间隔
- **THEN** 系统必须使用指定的间隔时间
- **AND** 必须支持单位（秒、分钟）
- **AND** 必须使用合理的默认值（30 秒）

#### Scenario: 配置检查超时

- **WHEN** 配置健康检查超时
- **THEN** 系统必须使用指定的超时时间
- **AND** 必须支持单位（秒）
- **AND** 必须使用合理的默认值（5 秒）

#### Scenario: 配置检查语句

- **WHEN** 配置健康检查语句
- **THEN** 系统必须使用指定的 SQL 语句
- **AND** 必须支持不同数据库类型的语句
- **AND** 必须使用默认语句（SELECT 1）

---

### Requirement: 健康状态查询

系统必须提供健康状态查询接口，允许获取实例的当前健康状态。

#### Scenario: 查询单个实例健康状态

- **WHEN** 请求单个实例的健康状态
- **THEN** 系统必须返回实例的当前健康状态
- **AND** 必须包含最后检查时间
- **AND** 必须包含检查结果详情

#### Scenario: 查询所有实例健康状态

- **WHEN** 请求所有实例的健康状态
- **THEN** 系统必须返回所有实例的健康状态
- **AND** 必须按健康状态分组
- **AND** 必须包含统计信息（健康/不健康实例数）

#### Scenario: 健康状态 API

- **WHEN** 调用健康检查 API 端点
- **THEN** 系统必须返回健康状态信息
- **AND** 必须支持 JSON 格式
- **AND** 必须包含详细的健康检查结果

---

### Requirement: 健康状态事件

系统必须在实例健康状态发生变化时触发事件。

#### Scenario: 实例变为健康

- **WHEN** 实例从 Unhealthy 变为 Healthy
- **THEN** 系统必须触发 InstanceHealthy 事件
- **AND** 必须记录状态变更日志
- **AND** 必须通知相关组件

#### Scenario: 实例变为不健康

- **WHEN** 实例从 Healthy 变为 Unhealthy
- **THEN** 系统必须触发 InstanceUnhealthy 事件
- **AND** 必须记录状态变更日志
- **AND** 必须通知相关组件
- **AND** 必须提供不健康的原因

#### Scenario: 实例状态未知

- **WHEN** 实例状态变为 Unknown
- **THEN** 系统必须触发 InstanceUnknown 事件
- **AND** 必须记录状态变更日志
- **AND** 必须通知相关组件

---

### Requirement: 健康检查指标

系统必须收集健康检查的指标，用于监控和告警。

#### Scenario: 健康检查指标收集

- **WHEN** 执行健康检查
- **THEN** 系统必须收集以下指标：
  - 健康检查执行次数
  - 健康检查成功次数
  - 健康检查失败次数
  - 平均检查时间
  - 最长检查时间
- **AND** 必须使用结构化日志格式

#### Scenario: 健康检查告警

- **WHEN** 实例连续多次检查失败
- **THEN** 系统必须触发告警
- **AND** 必须包含告警级别（警告/严重）
- **AND** 必须包含实例信息和失败原因

#### Scenario: 健康检查 Dashboard

- **WHEN** 访问健康检查 Dashboard
- **THEN** 系统必须显示以下信息：
  - 所有实例的健康状态
  - 健康检查历史趋势
  - 失败实例的详细信息
  - 告警历史
- **AND** 必须支持实时更新
