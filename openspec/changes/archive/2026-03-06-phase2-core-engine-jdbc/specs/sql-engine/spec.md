## ADDED Requirements

### Requirement: SQL 解析

系统必须能够解析 SQL 语句，识别语句类型和结构。

#### Scenario: 解析 SELECT 语句

- **WHEN** 输入 SELECT 语句
- **THEN** 系统必须识别为查询语句
- **AND** 必须提取表名、列名、WHERE 条件等信息
- **AND** 必须支持复杂的 SELECT 语句（JOIN、GROUP BY、ORDER BY）

#### Scenario: 解析 DML 语句

- **WHEN** 输入 INSERT/UPDATE/DELETE 语句
- **THEN** 系统必须识别为 DML 语句
- **AND** 必须提取操作类型、表名、条件等信息
- **AND** 必须支持批量操作

#### Scenario: 解析 DDL 语句

- **WHEN** 输入 CREATE/ALTER/DROP 语句
- **THEN** 系统必须识别为 DDL 语句
- **AND** 必须提取操作类型、对象类型、对象名等信息
- **AND** 必须标记为危险操作

#### Scenario: 解析存储过程和函数

- **WHEN** 输入 CALL 或 CREATE PROCEDURE 语句
- **THEN** 系统必须识别为存储过程相关语句
- **AND** 必须提取过程名和参数信息
- **AND** 必须支持存储过程调用

### Requirement: SQL 路由

系统必须根据实例名和 SQL 类型路由查询到正确的数据库实例。

#### Scenario: 实例路由

- **WHEN** 执行 SQL 查询
- **THEN** 系统必须根据指定的实例名路由到对应数据库
- **AND** 必须验证实例是否存在
- **AND** 必须处理实例不存在的情况

#### Scenario: 读写分离路由

- **WHEN** 执行 SELECT 语句
- **THEN** 系统必须尝试路由到从库
- **AND** 如果从库不可用，必须回退到主库
- **AND** 必须记录路由决策日志

#### Scenario: 跨实例查询

- **WHEN** 执行跨实例查询
- **THEN** 系统必须检测到跨实例操作
- **AND** 必须返回错误："不支持跨实例查询"
- **AND** 必须记录跨实例查询尝试

---

### Requirement: 查询超时控制

系统必须支持查询超时控制，防止长时间运行的查询。

#### Scenario: 设置查询超时

- **WHEN** 执行查询时设置超时时间
- **THEN** 系统必须使用指定的超时时间
- **AND** 必须在超时后取消查询
- **AND** 必须返回超时错误

#### Scenario: 默认超时设置

- **WHEN** 未设置查询超时
- **THEN** 系统必须使用默认超时时间（30 秒）
- **AND** 必须在配置中可调整
- **AND** 必须记录超时设置日志

#### Scenario: 超时处理

- **WHEN** 查询超过超时时间
- **THEN** 系统必须取消查询执行
- **AND** 必须释放相关资源
- **AND** 必须返回清晰的超时错误消息

---

### Requirement: 结果集大小限制

系统必须限制查询结果集大小，防止返回过多数据。

#### Scenario: 设置结果集限制

- **WHEN** 执行查询时设置结果集限制
- **THEN** 系统必须限制返回的行数
- **AND** 必须在响应中标记结果是否被截断
- **AND** 必须记录结果集限制日志

#### Scenario: 默认结果集限制

- **WHEN** 未设置结果集限制
- **THEN** 系统必须使用默认限制（1000 行）
- **AND** 必须在配置中可调整
- **AND** 必须记录默认限制使用日志

#### Scenario: 大结果集处理

- **WHEN** 查询结果超过限制
- **THEN** 系统必须只返回限制数量的行
- **AND** 必须在响应中包含截断提示
- **AND** 必须建议用户使用 LIMIT 子句

---

### Requirement: 表结构缓存

系统必须缓存表结构信息，提高查询性能。

#### Scenario: 表结构缓存初始化

- **WHEN** 首次访问表
- **THEN** 系统必须查询并缓存表结构
- **AND** 必须包含列名、类型、约束等信息
- **AND** 必须设置缓存过期时间

#### Scenario: 缓存命中

- **WHEN** 再次访问已缓存的表
- **THEN** 系统必须从缓存中获取表结构
- **AND** 必须验证缓存是否过期
- **AND** 必须记录缓存命中日志

#### Scenario: 缓存失效

- **WHEN** 表结构发生变化
- **THEN** 系统必须检测到变化
- **AND** 必须刷新缓存
- **AND** 必须记录缓存失效日志
