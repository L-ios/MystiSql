## Context

当前 MystiSql REPL 使用 `golang.org/x/term` 实现终端原始模式，`ReadLine` 结构体负责处理用户输入。现有实现存在以下限制：

1. **光标位置无追踪**: 输入只能追加到行尾，无法在行中间插入
2. **删除仅支持退格**: 只能删除最后一个字符，无法删除光标位置的字符
3. **方向键处理有限**: 仅处理上下键（历史浏览），忽略左右键
4. **无快捷键支持**: 缺少 Home/End、Ctrl+A/E 等常用快捷键

## Goals / Non-Goals

**Goals:**
- 实现完整的行编辑功能，支持光标在行内任意位置移动
- 支持在光标位置插入字符
- 支持 Delete 键删除光标位置字符
- 支持 Home/End 和 Ctrl+A/E 快速跳转
- 支持 Ctrl+U/K 删除到行首/行尾
- 保持与 MySQL 命令行客户端一致的用户体验

**Non-Goals:**
- 不实现多行编辑（上下移动光标跨越多行）
- 不实现复制粘贴功能（Ctrl+C 已用于中断）
- 不实现语法高亮（已有独立功能）
- 不引入新的外部依赖

## Decisions

### Decision 1: 光标位置追踪

**选择**: 在 `ReadLine` 结构体中添加 `cursorPos int` 字段

**理由**: 
- 简单直接，无需复杂数据结构
- 与现有 `[]rune` 存储方式兼容
- 便于计算屏幕刷新范围

**替代方案**:
- 使用双向链表：过于复杂，性能开销大
- 使用 rope 数据结构：对于单行编辑过于重量级

### Decision 2: 屏幕刷新策略

**选择**: 使用 ANSI 转义序列进行增量刷新

**理由**:
- 效率高，只刷新变化部分
- 标准 ANSI 序列，兼容性好
- 避免全行重绘带来的闪烁

**实现细节**:
```
光标左移: \b (backspace)
光标右移: \x1b[C (CSI C)
清除到行尾: \x1b[K (CSI K)
光标定位: \x1b[<n>G (CSI n G)
```

### Decision 2.5: 换行符处理

**选择**: 终端输出必须使用 `\r\n` 作为换行符

**理由**:
- **跨平台兼容性**: Windows 终端需要 `\r\n`，Unix/Linux 终端也能正确处理
- **光标定位准确**: 在终端原始模式下，`\n` 只会移动光标到下一行，不会回到行首
- **表格对齐**: 避免因换行符不一致导致的输出错位问题
- **与 MySQL CLI 一致**: MySQL CLI 在所有平台都使用正确的换行符

**实现要点**:
```go
// 正确的换行输出
fmt.Print("line 1\r\nline 2\r\n")

// 表格输出示例
fmt.Print("+----+-------+\r\n")
fmt.Print("| id | name  |\r\n")
fmt.Print("+----+-------+\r\n")

// 提示符换行
fmt.Print("mystisql@instance> ")
```

**注意事项**:
- 使用 `fmt.Print()` 或 `fmt.Fprintf()` 时手动添加 `\r\n`
- 避免使用 `fmt.Println()`，因为它在不同平台可能产生不同结果
- 在写入文件时可以继续使用 `\n`（文件不需要 `\r\n`）

### Decision 3: 输入处理架构

**选择**: 扩展现有 switch-case 结构，添加新的 case 分支

**理由**:
- 保持代码结构一致性
- 便于维护和测试
- 避免引入复杂状态机

### Decision 4: 快捷键映射

**选择**: 遵循 Emacs 风格快捷键（与 MySQL 一致）

| 快捷键 | 功能 |
|--------|------|
| ← / Ctrl+B | 光标左移一个字符 |
| → / Ctrl+F | 光标右移一个字符 |
| Home / Ctrl+A | 光标移到行首 |
| End / Ctrl+E | 光标移到行尾 |
| Alt+B | 向后移动一个词（backward-word） |
| Alt+F | 向前移动一个词（forward-word） |
| Delete | 删除光标位置字符 |
| Backspace | 删除光标前一个字符 |
| Ctrl+U | 删除从行首到光标位置 |
| Ctrl+K | 删除从光标位置到行尾 |

### Decision 5: 按词移动实现

**选择**: 使用空格作为词边界分隔符

**实现逻辑**:
```go
// Alt+B: 向后移动一个词
func (rl *ReadLine) moveCursorBackwardWord(line []rune) {
    // 1. 跳过当前位置的空格
    // 2. 找到前一个词的开头
    // 3. 移动光标到该位置
}

// Alt+F: 向前移动一个词
func (rl *ReadLine) moveCursorForwardWord(line []rune) {
    // 1. 跳过当前位置的空格
    // 2. 找到下一个词的结尾
    // 3. 移动光标到该位置
}
```

**理由**:
- 简单有效，覆盖大多数使用场景
- 与 MySQL CLI 行为一致
- 使用 `unicode.IsSpace()` 支持多种空白字符

**替代方案**:
- 使用正则表达式识别词边界：性能开销较大
- 支持自定义词边界：增加复杂度，初期不需要

## Risks / Trade-offs

### Risk 1: 终端兼容性
**风险**: 不同终端对 ANSI 转义序列支持可能不一致
**缓解**: 使用最基础的 ANSI 序列，已在主流终端测试通过

### Risk 2: 多字节字符处理
**风险**: 中文等多字节字符可能导致光标位置计算错误
**缓解**: 使用 `[]rune` 存储确保字符边界正确，屏幕宽度计算需考虑宽字符

### Risk 3: 性能影响
**风险**: 频繁刷新可能影响性能
**缓解**: 增量刷新策略，只更新必要部分

## Migration Plan

无需迁移，功能为纯增量添加，不影响现有行为。
