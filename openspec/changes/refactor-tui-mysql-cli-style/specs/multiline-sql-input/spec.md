## Purpose

定义 MystiSql REPL 中多行 SQL 输入功能的规范，支持用户输入跨越多行的 SQL 语句，提供与 MySQL CLI 一致的输入体验。

## ADDED Requirements

### Requirement: 分号结束符检测

系统必须正确识别 SQL 语句的结束位置。

#### Scenario: 单行语句结束

- **WHEN** 用户输入以分号结尾的单行 SQL
- **THEN** 系统必须识别该语句已完成
- **AND** 系统必须立即执行该语句

#### Scenario: 多行语句结束

- **WHEN** 用户在多行输入后输入分号
- **THEN** 系统必须识别整个多行语句已完成
- **AND** 系统必须将所有行合并后执行

#### Scenario: 分号在字符串内

- **WHEN** 分号出现在单引号或双引号字符串内
- **THEN** 系统必须不将其识别为语句结束符
- **AND** 系统必须继续等待更多输入

---

### Requirement: 续行提示符

系统必须提供清晰的多行输入状态提示。

#### Scenario: 标准续行

- **WHEN** 用户输入的 SQL 未以分号结尾
- **THEN** 系统必须显示续行提示符 `    ->`
- **AND** 系统必须等待用户继续输入

#### Scenario: 单引号字符串续行

- **WHEN** 用户输入包含未闭合的单引号字符串
- **THEN** 系统必须显示续行提示符 `    '>`
- **AND** 系统必须等待字符串闭合

#### Scenario: 双引号字符串续行

- **WHEN** 用户输入包含未闭合的双引号字符串
- **THEN** 系统必须显示续行提示符 `    ">`
- **AND** 系统必须等待字符串闭合

#### Scenario: 反引号标识符续行

- **WHEN** 用户输入包含未闭合的反引号标识符
- **THEN** 系统必须显示续行提示符 `    `>`
- **AND** 系统必须等待标识符闭合

---

### Requirement: 输入取消

系统必须支持取消当前输入。

#### Scenario: Ctrl+C 取消

- **WHEN** 用户在输入过程中按 Ctrl+C
- **THEN** 系统必须取消当前输入
- **AND** 系统必须显示 "^C" 提示
- **AND** 系统必须显示新的主提示符
- **AND** 系统不得执行已输入的内容

#### Scenario: 清空当前输入

- **WHEN** 用户取消输入后
- **THEN** 输入缓冲区必须被清空
- **AND** 系统必须准备好接收新的输入

---

### Requirement: 输入缓冲管理

系统必须正确管理多行输入缓冲。

#### Scenario: 行合并

- **WHEN** 用户完成多行输入
- **THEN** 系统必须将所有行合并为单个 SQL 语句
- **AND** 行之间必须用空格分隔
- **AND** 系统必须保留原始格式（缩进、空格）

#### Scenario: 大输入处理

- **WHEN** 用户输入超过 1MB 的 SQL 语句
- **THEN** 系统必须能够正常处理
- **AND** 系统不得崩溃或截断输入

---

### Requirement: 注释处理

系统必须正确处理 SQL 注释。

#### Scenario: 单行注释

- **WHEN** 用户输入包含 `--` 注释
- **THEN** 系统必须正确识别注释边界
- **AND** 注释后的分号必须被识别为结束符

#### Scenario: 多行注释

- **WHEN** 用户输入包含 `/* */` 多行注释
- **THEN** 系统必须正确识别注释边界
- **AND** 注释内的分号不得被识别为结束符

#### Scenario: 注释跨越多行

- **WHEN** 多行注释跨越多个输入行
- **THEN** 系统必须等待注释闭合
- **AND** 系统必须正确处理注释内的特殊字符
