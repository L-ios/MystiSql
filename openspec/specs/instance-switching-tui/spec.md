# TUI 实例切换规范

## Purpose

定义 MystiSql TUI 中数据库实例切换功能，包括实例列表显示、切换操作、连接管理和历史记录，确保用户可以方便地在多个数据库实例之间切换。

## Requirements

### Requirement: 实例列表显示

系统必须在 TUI 中提供实例列表功能。

#### Scenario: 查看所有实例

- **WHEN** 用户输入 `show instances` 命令
- **THEN** 系统必须显示所有可用实例
- **AND** 必须包含实例名称、类型（MySQL/PostgreSQL/Oracle/Redis）、状态
- **AND** 必须标记当前激活的实例（使用 * 或高亮显示）
- **AND** 实例必须按名称排序

#### Scenario: 按类型过滤实例

- **WHEN** 用户输入 `show instances --type mysql`
- **THEN** 系统必须只显示 MySQL 类型的实例
- **AND** 必须显示匹配的实例数量

#### Scenario: 显示实例详细信息

- **WHEN** 用户输入 `show instance <name>` 命令
- **THEN** 系统必须显示该实例的详细配置
- **AND** 必须包含主机地址、端口、数据库名
- **AND** 必须包含连接池状态（活跃连接数、空闲连接数）
- **AND** 必须包含最后连接时间

---

### Requirement: 实例切换操作

系统必须提供便捷的实例切换方式。

#### Scenario: 使用 Tab 键切换

- **WHEN** 用户按 Tab 键
- **THEN** 系统必须切换到列表中的下一个实例
- **AND** 提示符必须立即更新为新实例名称
- **AND** 必须显示切换成功消息
- **AND** 如果只有一个实例，必须显示提示信息

#### Scenario: 直接指定实例名称

- **WHEN** 用户输入 `use <instance-name>` 命令
- **THEN** 系统必须切换到指定实例
- **AND** 如果实例不存在，必须显示错误消息
- **AND** 提示符必须更新为 `mystisql@<instance-name>`

#### Scenario: 切换前测试连接

- **WHEN** 用户切换到新实例
- **THEN** 系统必须测试连接是否可用
- **AND** 如果连接失败，必须显示错误并保持在当前实例
- **AND** 如果连接成功，必须完成切换

#### Scenario: 显示连接状态

- **WHEN** 实例切换完成
- **THEN** 必须显示连接成功消息
- **AND** 必须显示连接建立时间（毫秒）
- **AND** 提示符必须反映新实例

---

### Requirement: 连接管理和重连

系统必须管理实例连接并支持自动重连。

#### Scenario: 自动重连机制

- **WHEN** 当前实例连接丢失
- **THEN** 系统必须自动尝试重连
- **AND** 必须显示"正在重连..."提示
- **AND** 最多重试 3 次，每次间隔 2 秒
- **AND** 如果重连失败，必须提示用户切换实例

#### Scenario: 连接池状态监控

- **WHEN** 用户查询实例状态
- **THEN** 系统必须显示连接池信息
- **AND** 必须包含：活跃连接数、空闲连接数、最大连接数
- **AND** 必须包含等待获取连接的请求数

#### Scenario: 连接超时处理

- **WHEN** 连接操作超过配置的超时时间
- **THEN** 系统必须取消连接尝试
- **AND** 必须显示超时错误消息
- **AND** 必须提示用户检查网络或增加超时时间

---

### Requirement: 实例健康状态

系统必须监控和显示实例健康状态。

#### Scenario: 健康检查

- **WHEN** TUI 启动或切换实例时
- **THEN** 系统必须对实例执行健康检查（如 SELECT 1）
- **AND** 必须标记实例状态为"健康"或"不健康"
- **AND** 实例列表必须用颜色标识状态（绿色=健康，红色=不健康）

#### Scenario: 不健康实例处理

- **WHEN** 实例被标记为不健康
- **THEN** 实例列表必须显示错误图标或标记
- **AND** 如果用户尝试切换到不健康实例，必须显示警告
- **AND** 必须允许用户强制切换

#### Scenario: 定期健康检查

- **WHEN** TUI 运行期间
- **THEN** 系统必须每隔 30 秒对所有实例执行健康检查
- **AND** 必须更新实例状态
- **AND** 不健康的实例必须在列表中突出显示

---

### Requirement: 切换历史和快速访问

系统必须维护实例切换历史。

#### Scenario: 记录切换历史

- **WHEN** 用户切换实例
- **THEN** 系统必须记录切换历史
- **AND** 历史记录必须包含：时间戳、实例名称
- **AND** 历史记录必须持久化到 ~/.mystisql_instance_history

#### Scenario: 快速切换到上一个实例

- **WHEN** 用户按 Ctrl+-
- **THEN** 系统必须切换到上一个使用的实例
- **AND** 必须显示切换消息

#### Scenario: 显示最近使用的实例

- **WHEN** 用户输入 `show recent-instances` 命令
- **THEN** 系统必须显示最近使用的 5 个实例
- **AND** 列表必须包含序号、实例名称、最后使用时间
- **AND** 用户可以输入序号快速切换

#### Scenario: 设置默认实例

- **WHEN** 用户输入 `set default-instance <name>` 命令
- **THEN** 系统必须将该实例设置为默认
- **AND** 下次启动 TUI 时必须自动连接到默认实例
- **AND** 必须显示确认消息

---

### Requirement: 实例分组管理

系统必须支持实例分组功能。

#### Scenario: 创建实例分组

- **WHEN** 用户输入 `create group <group-name> <instance1,instance2,...>` 命令
- **THEN** 系统必须创建新的实例分组
- **AND** 分组必须保存到配置文件
- **AND** 必须显示成功消息

#### Scenario: 按组列出实例

- **WHEN** 用户输入 `show instances --group <group-name>` 命令
- **THEN** 系统必须只显示该组内的实例
- **AND** 必须显示组名和实例数量

#### Scenario: 列出所有分组

- **WHEN** 用户输入 `show groups` 命令
- **THEN** 系统必须显示所有实例分组
- **AND** 每个分组必须显示名称和包含的实例数

---

### Requirement: 错误处理和提示

系统必须提供清晰的错误处理和用户提示。

#### Scenario: 实例不存在错误

- **WHEN** 用户尝试切换到不存在的实例
- **THEN** 系统必须显示错误："实例 '<name>' 不存在"
- **AND** 必须列出可用实例名称
- **AND** 必须保持在当前实例

#### Scenario: 所有实例不可用

- **WHEN** 所有配置的实例都无法连接
- **THEN** 系统必须显示严重错误消息
- **AND** 必须提示用户检查配置和网络
- **AND** 必须提供命令让用户重新加载配置

#### Scenario: 权限不足错误

- **WHEN** 用户尝试访问没有权限的实例
- **THEN** 系统必须显示权限错误
- **AND** 必须提示用户联系管理员
- **AND** 必须在实例列表中标记该实例为"无权限"
