# TUI SQL 执行规范

## Purpose

定义 MystiSql TUI 中 SQL 执行功能的详细规范，特别是针对长期运行的 SQL 查询、结果管理、批量执行和事务支持，确保用户可以在 TUI 中高效、安全地执行各种 SQL 操作。

## Requirements

### Requirement: 基本 SQL 执行

系统必须在 TUI 中提供基本的 SQL 执行功能。

#### Scenario: 执行单条 SQL 查询

- **WHEN** 用户输入 SELECT 语句并按 Enter
- **THEN** 系统必须执行查询
- **AND** 必须以表格格式显示结果
- **AND** 必须显示列名和数据行
- **AND** 必须显示总行数和执行时间（毫秒）

#### Scenario: 执行数据修改语句

- **WHEN** 用户执行 INSERT、UPDATE 或 DELETE 语句
- **THEN** 系统必须显示受影响的行数
- **AND** 对于 INSERT，必须显示最后插入的 ID
- **AND** 必须显示执行时间

#### Scenario: 执行 DDL 语句

- **WHEN** 用户执行 CREATE、ALTER、DROP 等语句
- **THEN** 系统必须显示执行结果（成功或失败）
- **AND** 如果成功，必须显示"查询成功"消息
- **AND** 必须显示执行时间

#### Scenario: 处理空结果集

- **WHEN** 查询返回空结果
- **THEN** 系统必须显示"Empty set"消息
- **AND** 必须显示执行时间
- **AND** 必须立即返回提示符

---

### Requirement: 长期运行查询处理

系统必须正确处理长期运行的 SQL 查询。

#### Scenario: 显示执行进度

- **WHEN** 查询执行时间超过 1 秒
- **THEN** 系统必须显示进度指示器（旋转动画或进度条）
- **AND** 必须显示已经执行的时长
- **AND** 必须在查询完成后立即显示结果

#### Scenario: 异步执行不阻塞界面

- **WHEN** 查询正在执行
- **THEN** TUI 界面必须保持响应
- **AND** 用户必须能够按 Ctrl+C 取消查询
- **AND** 必须显示"查询正在执行..."提示

#### Scenario: 取消执行中的查询

- **WHEN** 用户按 Ctrl+C 取消执行中的查询
- **THEN** 系统必须终止查询执行
- **AND** 必须显示"查询已取消"消息
- **AND** 必须释放数据库连接
- **AND** 必须返回提示符

#### Scenario: 查询超时处理

- **WHEN** 查询执行时间超过配置的超时时间（默认 30 秒）
- **THEN** 系统必须自动取消查询
- **AND** 必须显示超时错误消息
- **AND** 必须建议用户优化查询或增加超时时间

---

### Requirement: 结果显示和分页

系统必须支持大量结果的分页显示。

#### Scenario: 自动分页显示

- **WHEN** 查询结果超过终端高度
- **THEN** 系统必须分页显示结果
- **AND** 必须在底部显示"-- More --"提示
- **AND** 用户按空格键必须显示下一页
- **AND** 用户按 q 键必须退出分页模式

#### Scenario: 水平滚动显示

- **WHEN** 查询结果列数过多超出屏幕宽度
- **THEN** 系统必须支持水平滚动
- **AND** 用户按左右箭头键必须滚动列
- **AND** 必须显示当前可见列的范围（如列 5-10/20）

#### Scenario: 显示结果统计

- **WHEN** 查询结果完全显示
- **THEN** 系统必须显示统计信息
- **AND** 必须包含：总行数、总列数
- **AND** 必须包含执行时间
- **AND** 如果结果被截断，必须显示总可用行数

---

### Requirement: 结果排序和过滤

系统必须支持对查询结果进行排序和过滤。

#### Scenario: 按列排序

- **WHEN** 用户输入 `ORDER BY <column>` 命令
- **THEN** 系统必须对当前结果集按指定列排序
- **AND** 必须支持 ASC（升序）和 DESC（降序）
- **AND** 必须更新显示的结果

#### Scenario: 过滤结果

- **WHEN** 用户输入 `FILTER <column> <operator> <value>` 命令
- **THEN** 系统必须过滤当前结果集
- **AND** 必须只显示符合条件的行
- **AND** 必须显示过滤后的行数

#### Scenario: 在结果中搜索

- **WHEN** 用户按 / 键并输入搜索词
- **THEN** 系统必须在结果中高亮显示匹配项
- **AND** 用户按 n 键必须跳转到下一个匹配项
- **AND** 用户按 N 键必须跳转到上一个匹配项

---

### Requirement: 结果导出

系统必须支持导出查询结果。

#### Scenario: 导出为 CSV 格式

- **WHEN** 用户输入 `export result.csv csv` 命令或按 Ctrl+E 选择 CSV
- **THEN** 系统必须将最后的查询结果保存为 CSV 文件
- **AND** CSV 必须包含列名作为第一行
- **AND** 必须正确处理特殊字符（使用引号）
- **AND** 必须显示成功消息和文件路径

#### Scenario: 导出为 JSON 格式

- **WHEN** 用户输入 `export result.json json` 命令
- **THEN** 系统必须将结果保存为 JSON 数组格式
- **AND** 每行必须是独立的 JSON 对象
- **AND** 必须包含所有列名作为键

#### Scenario: 导出前确认

- **WHEN** 用户执行导出命令
- **THEN** 系统必须显示将要导出的行数
- **AND** 如果结果集很大（>10000 行），必须请求确认
- **AND** 用户可以取消导出操作

#### Scenario: 导出失败处理

- **WHEN** 导出失败（如磁盘空间不足、权限不足）
- **THEN** 系统必须显示详细的错误消息
- **AND** 必须不创建部分文件
- **AND** 必须建议用户检查磁盘空间或权限

---

### Requirement: 批量 SQL 执行

系统必须支持批量执行多条 SQL 语句。

#### Scenario: 执行多条语句

- **WHEN** 用户粘贴或输入多条 SQL 语句（用分号分隔）
- **THEN** 系统必须按顺序执行每条语句
- **AND** 必须为每条语句显示执行结果
- **AND** 必须显示总体执行统计（成功数、失败数）

#### Scenario: 遇到错误停止

- **WHEN** 批量执行中某条语句失败
- **THEN** 系统必须停止后续语句的执行
- **AND** 必须显示失败的语句和错误信息
- **AND** 必须显示已成功执行的语句数量

#### Scenario: 忽略错误继续执行

- **WHEN** 用户输入 `SET batch_continue_on_error=true` 后执行批量语句
- **THEN** 系统必须继续执行后续语句即使某条失败
- **AND** 必须在最后显示所有失败语句的摘要
- **AND** 必须显示成功和失败的计数

#### Scenario: 从文件执行批量 SQL

- **WHEN** 用户输入 `source /path/to/script.sql` 命令
- **THEN** 系统必须读取文件中的 SQL 语句
- **AND** 必须逐条执行所有语句
- **AND** 必须显示每条语句的执行结果
- **AND** 文件必须支持注释（以 -- 或 # 开头的行）

---

### Requirement: 事务管理

系统必须在 TUI 中支持事务操作。

#### Scenario: 开始事务

- **WHEN** 用户输入 `BEGIN` 或 `START TRANSACTION`
- **THEN** 系统必须开始一个新事务
- **AND** 必须显示"事务已开始"消息
- **AND** 提示符必须显示事务标记（如 `[TX]`）

#### Scenario: 提交事务

- **WHEN** 用户在事务中输入 `COMMIT`
- **THEN** 系统必须提交当前事务
- **AND** 必须显示"事务已提交"消息
- **AND** 必须清除提示符中的事务标记

#### Scenario: 回滚事务

- **WHEN** 用户在事务中输入 `ROLLBACK`
- **THEN** 系统必须回滚当前事务
- **AND** 必须显示"事务已回滚"消息
- **AND** 必须清除提示符中的事务标记

#### Scenario: 显示事务状态

- **WHEN** 用户输入 `SHOW TRANSACTION STATUS` 命令
- **THEN** 系统必须显示当前是否有活跃事务
- **AND** 必须显示事务开始时间
- **AND** 必须显示事务隔离级别

#### Scenario: 设置隔离级别

- **WHEN** 用户输入 `SET TRANSACTION ISOLATION LEVEL <level>`
- **THEN** 系统必须设置事务隔离级别
- **AND** 必须显示确认消息
- **AND** 支持的级别必须包括：READ COMMITTED, REPEATABLE READ, SERIALIZABLE

#### Scenario: 事务超时警告

- **WHEN** 事务持续时间超过 60 秒
- **THEN** 系统必须显示警告消息
- **AND** 必须建议用户提交或回滚事务
- **AND** 必须在状态栏显示事务持续时间

---

### Requirement: 查询性能优化

系统必须提供查询性能优化辅助功能。

#### Scenario: 显示执行计划

- **WHEN** 用户在 SQL 前加 `EXPLAIN` 或输入 `EXPLAIN <sql>`
- **THEN** 系统必须显示查询执行计划
- **AND** 必须以表格格式显示
- **AND** 必须包含：操作类型、表名、索引使用情况等

#### Scenario: 查询性能警告

- **WHEN** 查询执行时间超过 5 秒
- **THEN** 系统必须显示性能警告
- **AND** 必须建议用户使用 EXPLAIN 分析查询
- **AND** 必须建议检查索引和 WHERE 条件

#### Scenario: 查询缓存

- **WHEN** 用户执行相同的查询两次
- **THEN** 系统可以使用缓存的连接
- **AND** 不得缓存查询结果（确保数据实时性）
- **AND** 必须显示每次查询的实际执行时间

---

### Requirement: SQL 执行错误处理

系统必须提供详细的错误处理和诊断信息。

#### Scenario: 语法错误提示

- **WHEN** SQL 语句有语法错误
- **THEN** 系统必须显示错误位置（行号、字符位置）
- **AND** 必须高亮显示错误的部分
- **AND** 必须显示数据库返回的错误消息
- **AND** 必须提供修正建议（如果可能）

#### Scenario: 连接丢失处理

- **WHEN** 执行 SQL 时数据库连接丢失
- **THEN** 系统必须显示"连接丢失"错误
- **AND** 必须自动尝试重连
- **AND** 如果重连成功，必须询问用户是否重新执行 SQL

#### Scenario: 权限错误处理

- **WHEN** 用户执行没有权限的操作
- **THEN** 系统必须显示权限错误消息
- **AND** 必须显示缺少的具体权限
- **AND** 必须建议用户联系管理员获取权限

#### Scenario: 资源限制错误

- **WHEN** 查询结果超过内存限制或行数限制
- **THEN** 系统必须显示资源限制错误
- **AND** 必须建议用户使用 LIMIT 子句
- **AND** 必须建议使用分页查询
